# 商品导入说明书

## 你以后只需要做什么

准备一个“商品包”文件夹，然后运行一次导入命令并选择类目。工具会自动建立产品目录、更新前端菜单、首页无 JavaScript 商品入口和静态分享页；**不再需要手动修改 `app.js`、首页或商品分享页**。

这是一个静态网站：导入器运行在你的 Mac 本机，不是网站前台的上传按钮。导入完成也不会自动发布，仍需要你确认 Git 变更后再提交和推送。

## 前提

- 已安装 Node.js 20 或更高版本和 Git。
- 在仓库根目录运行命令：`/Users/tylertang/Developer/ai-coding/3D-RedWood`。
- 不要直接双击 `index.html`。本地预览请使用 `npm run serve`。

## 商品包怎么准备

一个商品对应一个独立文件夹，最简单的形式如下：

```text
我的新商品/
  model.glb                 # 必填：3D 模型
  info.json                 # 推荐：商品信息
```

`info.json` 示例：

```json
{
  "name": "紫檀祥云圆底座",
  "price": "¥ 88",
  "description": "适合茶室、书房与文玩陈列的手工红木底座。",
  "id": "zitan_xiangyun"
}
```

- `name`：必填，1–80 个字符。
- `price`：必填，1–40 个字符，保留你希望展示的文字，例如 `¥ 88` 或 `¥ 88 - ¥ 128`。
- `description`：必填，1–3000 个字符；可换行。
- `id`：可选。商品的稳定目录 ID，只能是小写字母、数字、下划线，例如 `zitan_xiangyun`。没有填写时，导入器会询问你。

不要把 HTML、脚本或其他附件放进商品包；导入器只会复制 `model.glb`，并生成自己的 `info.json` 和商品页。

## 最快的导入方法（推荐）

在仓库根目录执行：

```bash
npm run add-product -- --source "/商品包的完整路径"
```

例如：

```bash
npm run add-product -- --source "/Users/your-name/Desktop/我的新商品"
```

随后按提示操作：

1. 输入现有类目的编号；输入 `new` 可新建类目。
2. 新类目需要填写稳定的类目 ID（例如 `yuanxing_dizuo`）和显示名称（例如“圆形底座”）。
3. 输入商品 ID；`info.json` 已有名称、价格、简介时可直接按 Enter 使用。
4. 等待工具复制、修复并校验模型，随后自动运行 `npm test`。
5. 看到“导入完成”后，检查 Git 变更，再决定是否提交发布。

一个商品在当前版本只属于一个类目。已有同名商品 ID 或已有目标目录时，导入器会拒绝覆盖。

## 全自动导入（可选）

如果商品信息已经准备齐全，可以不用交互输入：

```bash
npm run add-product -- \
  --source "/Users/your-name/Desktop/我的新商品" \
  --category yuanxing_dizuo \
  --product-id zitan_xiangyun \
  --name "紫檀祥云圆底座" \
  --price "¥ 88" \
  --description "适合茶室、书房与文玩陈列的手工红木底座。"
```

商品包已有完整 `info.json` 时，仍需要 `--category` 和 `--product-id`；名称、价格和简介可以省略。

## 工具会自动改哪些内容

导入一个商品会生成或更新：

```text
products/<类目 ID>/<商品 ID>/model.glb
products/<类目 ID>/<商品 ID>/info.json
products/<类目 ID>/<商品 ID>/index.html
products/catalog.json
index.html
```

`products/catalog.json` 是唯一商品清单来源。前端菜单、无 JavaScript 的商品入口和商品分享页都由它生成或读取。

## 导入后的检查与发布

导入器已经会执行 `npm test`。仍建议在发布前执行：

```bash
npm test
npm run serve
git diff --check
git status --short
```

打开 `http://127.0.0.1:4173`，从分类菜单点击新商品，再直接打开：

```text
http://127.0.0.1:4173/?product=<类目 ID>/<商品 ID>
```

确认模型、标题、价格、介绍和静态商品页都正确后，再提交与推送：

```bash
git add products/catalog.json products/<类目 ID>/<商品 ID> index.html
git add app.js app-core.js scripts tests docs package.json
git commit -m "feat: add <商品名称>"
git push
```

第一次上线这一套导入器时，第二行会把工具本身一起提交；以后只需提交新商品目录、`products/catalog.json` 和 `index.html`。

## 常见错误

| 提示 | 含义和处理 |
| --- | --- |
| `WebP textures` | 重新导出模型，将贴图改为 PNG 或 JPEG。 |
| `must include TANGENT data` | 模型使用了法线贴图，但缺少切线；请让建模软件导出 Tangent，或移除法线贴图后重新导出。 |
| `exceeds the 50 MB import limit` | 压缩/优化模型和贴图后再导入。 |
| `already exists` | 商品 ID 已占用；换一个 ID，或另行处理原商品，导入器不会覆盖。 |
| `Invalid product ...` | 检查名称、价格、简介是否为空或过长；ID 只能使用小写字母、数字、下划线。 |
| `file://` 页面无法加载信息 | 不要双击 HTML，改用 `npm run serve`。 |

## 失败与回滚

- 模型校验、目录生成或 `npm test` 失败时：本次新增目录、目录清单和首页会自动恢复；原始商品包不会被改动。
- 导入成功但还没提交时：先用 `git diff` 检查。若确认不要该商品，只删除本次新增的准确目录，并恢复两个生成文件：

```bash
git restore products/catalog.json index.html
```

然后在 Finder 中移除本次新增的 `products/<类目 ID>/<商品 ID>/` 目录，或明确指定该目录后删除。不要使用宽泛的递归删除命令。

- 已经提交后：使用 `git revert <提交 ID>` 生成可追溯的回滚提交。
